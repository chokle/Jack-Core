---
name: Trust-driven retrieval reranking
description: How reviewer verify/reject decisions AND corroboration confidence on distilled concepts influence Ask Jack + semantic search
---
Retrieval (search + chat) runs over `transcript_segments` via pgvector; those rows have no trust signal. Trust lives on `knowledge_nodes` (`verification_status`, derived noisy-OR `confidence`, and `meta.sourceCount`). The only bridge is `artifacts/api-server/src/lib/verification-rerank.ts`: it maps a concept's `meta.sources[{videoId,timestamps}]` onto a retrieved segment by (videoId + time-window ±2s) overlap, then: drops rejected-only segments, boosts verified-covered segments (+0.15), and additionally adds a corroboration bump (≤0.1, `corroborationBoost(confidence, sourceCount)` — zero for a single source, saturates at 3 sources, scaled by confidence). Precedence per segment: verified > rejected > unverified-corroborated > neutral. Verified wins over rejected; a reviewer rejection wins over automatic corroboration.

**Why:** originally only binary verify/reject steered RAG; corroboration (many videos + confidence) was added so a one-off mention no longer ranks equal to a well-confirmed, mentor-verified concept.

**How to apply:** coverage now includes UNVERIFIED concepts too (any non-verified/rejected status collapses to `unverified`), so `fetchVerificationCoverage` no longer filters status in SQL — it resolves concept ids from provenance edges (`kind='knowledge'`, `source_id='video:<id>'`) scoped to the retrieved videos, then reads those nodes. This keeps the fetch bounded (do NOT revert to "select all knowledge_nodes" — that scans the whole graph). chat.ts keeps the full `rerankByVerification` result (not just `.item`) and passes `{verification, sourceCount}` to `describeTrust()` which appends a `· mentor-verified · confirmed across N videos` tag into the context so Jack can cite trust; the jurisdiction prompt tells Jack to prefer higher-trust sources. Concepts whose sources have empty `timestamps` are ignored (never boost/suppress a whole video off one node). Coverage-load DB failures degrade to un-reranked retrieval; trust is an enhancement, never a hard dependency of answering. `knowledge_entries` (field notes) carry no trust and are out of scope.
