# Sprint: Jack Vital Signs v1

Objective:
Transform the current heartbeat from a decorative animation into a live telemetry-driven system that reflects Jack's actual operating state.

## Current Status

The existing implementation is an excellent foundation. We are now evolving it into a first-class system component.

---

## UI Refinements

Increase the ECG line length by approximately 25–35%.

The ECG should feel like a real patient monitor rather than a small icon.

Current:

❤️ ╲╱──

Target:

❤️ ───╲╱──────╲╱────────╲╱────

The animation must remain smooth and subtle.

No flashy effects.

---

## Layout

Separate the status from the BPM.

Instead of:

HEALTHY 65 BPM

Use:

HEALTHY
65 BPM

or

● HEALTHY
65 BPM

Increase readability and visual hierarchy.

---

## Heart

The heart icon remains permanently red.

It is Jack's identity.

It should never change color based on state.

Only the ECG pulse changes.

---

## Pulse States

The ECG color should represent Jack's operating mode.

Green
- Healthy
- Idle

Purple
- Reasoning

Orange
- Learning
- Memory Writes

Red
- Warning / Error

Avoid rapidly cycling colors.

The pulse should remain in one state for the duration of an operation.

---

## Build a Vitality Engine

Create a reusable service responsible for all heartbeat behavior.

Responsibilities:

- heartbeatBPM
- pulseColor
- systemStatus
- vitalityScore (0–100)

Future systems must publish events to this service instead of directly manipulating the UI.

The UI should subscribe only to the Vitality Engine.

---

## Telemetry Inputs

Use real runtime information whenever available.

Suggested event sources:

- LLM inference start/end
- Knowledge graph search
- Memory write events
- Embedding generation
- Queue depth
- Active API requests
- CPU utilization
- RAM utilization
- Background jobs
- Connector activity

---

## BPM Logic

Heartbeat should never appear random.

Suggested ranges:

Sleeping
58–62 BPM

Idle
65–70 BPM

Conversation
72–78 BPM

Reasoning
80–88 BPM

Heavy ingestion
90–95 BPM

Recovery back to resting should be gradual over several seconds.

---

## ECG Behavior

The ECG should scroll continuously.

Speed scales with BPM.

Every 6–10 seconds introduce a slightly stronger heartbeat.

Not larger.

Not brighter.

Simply a slightly different rhythm so it feels organic rather than generated by a repeating CSS animation.

---

## Future Extensibility

Design this as a platform service.

Future subsystems should be able to publish events such as:

publish("reasoning_start")
publish("memory_write")
publish("graph_search")
publish("reasoning_complete")

without requiring UI changes.

---

## Goal

Jack should no longer feel like a webpage.

He should feel like a continuously running intelligence engine whose internal state is visible at a glance.

Think industrial telemetry.

Think aircraft instrumentation.

Think mission control.

Minimal.

Elegant.

Purpose-built.
